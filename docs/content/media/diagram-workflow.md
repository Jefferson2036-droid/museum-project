# Diagram Workflow

## Purpose

This document defines how explanatory diagrams are planned and recorded for the
AI history site.

## First-Supported Diagram Types

Sprint 2 supports diagrams that clarify:

- chronology
- conceptual relationships
- causal or institutional connections

Mermaid is the default planning format when a simple text-based diagram is
enough.

## Diagram Planning Rules

- a diagram must teach one clear idea
- it should reduce confusion that prose alone would leave behind
- it should attach to a specific era, concept, or turning point
- it should not duplicate a guide callout word for word
- if the diagram competes with the chronology instead of supporting it, it is a
  bad candidate

## What To Record In A Diagram Plan

Each diagram record should include:

- the teaching question it answers
- linked content entities
- the specific relationship or sequence it should show
- a draft node list or flow
- why a diagram is better than prose alone for this case

## Exemplar Diagram Direction

The first required exemplar plan is:

- `logic-to-ai-diagram.md`

This diagram should support the Era 2 throughline by showing that multiple
lines of work converged before Dartmouth named the field.

## Diagram Scope Discipline

- keep the first diagrams narrow enough to fit the exemplar slice
- avoid trying to diagram the whole history of AI in one artifact
- use diagrams to slow down dense transitions, especially where chronology and
  concept teaching overlap
