# Image Workflows

## Purpose

This document defines how sourced-image and generated-image candidates are
discovered, evaluated, and recorded for the AI history site.

## Core Rule

Images must support comprehension and historical specificity.

They are not decorative filler and they are not a substitute for narrative or
guide commentary.

## Sourced Images

Use sourced images when they provide real historical value, such as:

- a primary document or historically meaningful object
- a person, lab, device, or artifact directly tied to the story
- an image that helps the reader locate a turning point in time or context

Every sourced-image candidate should record:

- what the image depicts
- source URL or origin
- source or creator name when known
- how it connects to the era, entity, or turning point
- attribution requirements when known
- why the image improves teaching instead of merely illustrating a mood

## Generated Images

Use generated images only when one of these conditions is true:

- sourced media is unavailable
- sourced media is too weak, unclear, or generic to teach well
- an explanatory visual would be more pedagogically useful than an archival
  image

Every generated-image candidate should record:

- what problem the generated visual solves
- why sourced media is insufficient
- the historical accuracy constraints the image must respect
- the prompt or generation brief when available
- any note explaining that the asset is an original explanatory image rather
  than archival media

## Selection Rules

- prefer sourced images for primary historical anchors
- prefer generated images for explanatory visuals that clarify relationships or
  abstractions
- reject any image that competes with the narrative, confuses the time period,
  or introduces a false sense of specificity

## Rejected Image Candidates

If an image candidate is rejected and the reason is likely to matter later,
record the rejection briefly in the asset record.

Useful rejection reasons include:

- unclear provenance
- weak teaching value
- generic or misleading imagery
- attribution uncertainty that blocks responsible use

Do not create a separate rejection log for every low-quality idea.

## Sprint 2 Exemplar Direction

The first sourced-image records should come from the Era 2 exemplar slice:

- Dartmouth proposal image
- Shannon / Theseus imagery from Bell Labs historical material

Sprint 2 does not require a generated-image record yet, but the workflow must
make the distinction explicit so later sprints can add explanatory visuals
cleanly.
